# Contributing to SnapFlow

We love your input! We want to make contributing to SnapFlow as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Contributor License Agreement

By contributing to this project, you agree to the terms of the [Contributor License Agreement (CLA)](CLA.md). All contributions will be assigned to the Project Owner to ensure the long-term sustainability and legal protection of the software.

## Our Development Process

We use GitHub to track issues and pull requests.

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!
